Guide UX Lab

A code-first prototyping initiative that replaced click-through mockups with a working product artifact for planning, GTM alignment, and engineering handoff.

Role: Senior Product Designer Timeframe: 2024 to 2025 Tools: VS Code, Next.js, TypeScript, AI-assisted workflow

Featured Work Summary

Designing to: reduce ambiguity between design and engineering in complex platform workflows.

Code-First Prototype GTM Enablement Developer-Facing Platform Strategy

Executive Summary

Context

The team needed a realistic way to validate feature behavior before full implementation and before GTM planning locked.

Role

I led design and built the prototype directly in code, while coordinating product and engineering feedback loops.

Challenge

Click-through prototypes communicated layout but left important interaction and state behavior open to interpretation.

Objective

Reduce handoff ambiguity by using a runnable artifact that made decisions, edge cases, and tradeoffs explicit.

Outcome

The Lab improved cross-functional alignment and gave teams a shared reference for implementation and launch planning.

Problem / Context

Traditional handoff artifacts were useful for visual review but weak for behavior review. Questions around loading states, edge cases, navigation logic, and system rules often surfaced late, which slowed planning and increased translation effort between design and engineering.

We needed a process that preserved design intent while allowing engineering-grade validation earlier in the PDLC.

Constraints

  • Prototype fidelity had to be high enough for planning decisions, not only visual direction.
  • The approach needed to support collaboration with product, engineering, and GTM teams in parallel.
  • Patterns had to be reusable so the prototype could inform future platform work instead of one release only.

Approach / Strategy

The Lab used a code-first strategy: interaction and state behavior were authored directly in a production-ready stack. AI support was used for rapid iteration, scenario expansion, and test coverage of edge states. This allowed design and engineering to review the same artifact instead of translating between separate prototype formats.

Architecture / Flows

Key Patterns / Decisions

  • Build interaction-critical flows in code first, then layer visual refinement.
  • Define component APIs and state variants early to reduce interpretation drift.
  • Use one shared artifact for design review, engineering planning, and GTM walkthroughs.

Collaboration

Product, design, engineering, and GTM partners reviewed the same runnable prototype throughout planning. This reduced ambiguity in scope discussions and improved confidence in launch sequencing because behavior-level details were visible early.

Outcomes

  • Reduced handoff ambiguity across core workflow decisions.
  • Improved cross-functional alignment during GTM and release planning.
  • Established a repeatable AI-assisted, code-first pattern for future PDLC work.

Reflection / Learnings

Code-first prototyping works best when it is treated as a collaboration artifact, not a solo design exercise. The largest gain was reducing interpretation gaps early, which improved planning quality for both product and engineering.
